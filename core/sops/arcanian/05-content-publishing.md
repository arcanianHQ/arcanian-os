# SOP-05: Content Publishing

> Content creation and publishing workflow for LinkedIn and Substack.

---

## 1. Purpose

Maintain a consistent content cadence that builds Arcanian's market presence and ensures every piece of content lands on marketing system diagnosis. Content is not self-promotion -- it documents real work, shares real learnings, and demonstrates the diagnostic methodology in action.

**Core rule:** Every post must give real value. Document real work, share learnings. Never self-promotional.

## 2. Trigger

- Content Calendar date (Wednesday or Friday)
- Significant client insight or methodology evolution worth documenting
- External event/conversation that connects to marketing system diagnosis

## 5. Steps

### 5.1 Content Calendar Check

| Day | Format | Description |
|-----|--------|-------------|
| Wednesday | **Pattern Drop** | Diagnostic insight, system-aware. AI hook allowed as on-ramp. |
| Friday | **Quality Bomb** | Personal story, self-diagnostic. Vulnerability = trust. |

- Check `CONTENT_CALENDAR_2026.md` for the week's assigned topics
- If no assigned topic: pick from backlog or document a recent client/methodology insight

### 5.2 Bridge Rule Enforcement

**Every post can START anywhere. Every post must LAND on marketing system diagnosis.**

- Pattern Drop (Wed): AI/industry hook --> turn --> L0-L7 layer insight
- Quality Bomb (Fri): personal story --> turn --> "this is why I diagnose which layer is broken"
- AI content = on-ramp (credibility). Marketing diagnosis = destination (L2 decoded correctly).
- Before finalizing: ask -- "If someone reads only this post, what do they decode about Arcanian?" Answer must include "diagnoses marketing systems" or equivalent.

### 5.3 Draft in Claude Code

- Write draft following the assigned format
- Include the "turn" moment explicitly -- where does the post pivot from hook to diagnostic insight?
- Keep voice: honest practitioner, not expert from above
- NEVER write "ooszinten" (honestly), "nem felulrol" (not from above), or "segitek" (I help)
- Show don't tell: vulnerability and honesty must be FELT through tone

### 5.4 Voice Check

- Research Lead reviews draft against `brand/VOICE.md`
- Check: does it sound like an honest practitioner or a guru?
- Check: any forbidden phrases?
- Check: does the Bridge Rule land?

### 5.5 Publish

- Publish to LinkedIn (primary) and/or Substack (long-form)
- Ensure formatting is clean on the platform
- Tag relevant people only if genuinely relevant (no engagement farming)

### 5.6 Update PUBLICATION_DIRECTORY.md

- Log: date, title, platform, format (Pattern Drop / Quality Bomb), link
- Note: Bridge Rule compliance (yes/no)
- Note: any notable engagement or reactions

### 5.7 Track Engagement

- Monitor comments for references to "reteg" (layer), "diagnozis" (diagnosis), "rendszer" (system)
- These terms in comments = Bridge Rule working (audience decodes correctly)
- Track monthly: what % of comments reference diagnostic/system language?

## 6. Escalation

- If content cadence drops below 2 posts/week for 2+ consecutive weeks --> Founder reviews priorities and capacity
- If Bridge Rule compliance drops below 75% --> Team review: are we drifting back to "AI philosopher" encoding?
- If engagement drops significantly --> Review: is the content giving real value or becoming formulaic?

## 7. KPIs

| Metric | Target |
|--------|--------|
| Posts per week | >= 2 (Wed + Fri minimum) |
| Bridge Rule compliance | >= 75% of posts land on marketing diagnosis |
| PUBLICATION_DIRECTORY updated | After every post |
| Monthly comment tracking | Completed (% referencing diagnostic language) |
| Voice check completed | Before every publish |

## 8. Review Cadence

- **Weekly:** Quick check -- did both posts go out? Bridge Rule compliance?
- **Monthly:** Comment language analysis -- are we being decoded as "fractional CMO who diagnoses systems" or something else?
- **Quarterly:** Content strategy review -- update Content Calendar, refresh topic backlog
