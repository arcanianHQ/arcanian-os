# 7-Layer Diagnostic Scoring Rubrics

## Purpose

Each agent diagnoses a pair of layers. The council identifies the PRIMARY CONSTRAINT — the deepest layer blocking growth — and maps how problems cascade.

## The Direction Rule

**Diagnose INSIDE OUT (L0→L7), fix INSIDE FIRST.**

Problems at deeper layers (L0-L2) make surface-layer fixes (L5-L7) ineffective. A broken L2 (identity) means L5 (channel) optimization is wasted spend.

## Layer Health Scoring

Each layer is scored 0-100:

| Score | Status | Meaning |
|---|---|---|
| ≥80 | Healthy | Layer is functioning, not a constraint |
| 60-79 | Stressed | Some issues, may be contributing to problems |
| 40-59 | Constrained | Significant issues, likely blocking growth |
| <40 | Broken | Primary constraint candidate — fix this first |

## Aggregation

The 7-layer council does NOT produce a single weighted score. It produces:

1. **Layer health table** — all 8 layers scored 0-100
2. **PRIMARY CONSTRAINT** — the deepest broken layer (lowest score in L0-L3 range)
3. **CASCADE MAP** — how the constraint cascades up through higher layers
4. **Direction rule validation** — are they fixing the right layer?

**Primary Constraint Rule:** Always look deepest first. If L2 is broken AND L5 is broken, the primary constraint is L2 (deeper). Fixing L5 without fixing L2 = wasted effort.
