# Skill: OKR-BLUF Dokumentum Készítés

## Mikor használd?

Amikor az ügyfélnek stratégiai áttekintő dokumentumot kell készíteni:
- Projekt státusz összefoglaló
- Marketing/üzleti terv
- Transcript elemzés eredménye
- Quarterly review

---

## Dokumentum Struktúra

### 1. BLUF (Bottom Line Up Front)

**Mindig a dokumentum elején!** A vezető/ügyfél azonnal lássa a lényeget.

```markdown
## BLUF (Bottom Line Up Front)

### Mi a helyzet most?
[1-2 mondat a jelenlegi státuszról]

### Mi kell MOST?
| Prioritás | Teendő | Felelős | Blokkoló? |
|-----------|--------|---------|-----------|
| 🔴 | [Kritikus] | [Név] | [Igen/Nem] |
| 🟡 | [Fontos] | [Név] | - |
| 🟢 | [Normál] | [Név] | - |

### Főbb eredmények eddig
✅ [Kész dolog 1]
✅ [Kész dolog 2]
🔄 [Folyamatban lévő]

### Fő üzleti célok
| Cél | Metrika |
|-----|---------|
| [Cél 1] | [Szám/KPI] |

### Kulcs üzenet
> [1 mondat - a legfontosabb takeaway]
```

---

### 2. OKR Struktúra

**Objective → Key Results → Initiatives**

```markdown
## O1: [Objective neve]

> **Státusz:** 🔄 In Progress | ⏳ Tervezés | ✅ Kész

### Key Results

| KR | Cél | Jelenlegi | Státusz |
|----|-----|-----------|---------|
| KR1.1 | [Mérhető eredmény] | [Aktuális érték] | 🔄/⏳/✅ |

### Initiatives

#### I1.1 [Initiative neve]

> **Státusz:** [emoji + szöveg]

**Leírás/Részletek:**
- Pont 1
- Pont 2

**Következő lépések:**
1. [Lépés]
2. [Lépés]
```

---

### 3. Státusz Jelölések

| Emoji | Jelentés |
|-------|----------|
| ✅ | Kész / Megoldva |
| 🔄 | In Progress |
| ⏳ | Tervezés / Várakozik |
| 🔴 | Kritikus prioritás |
| 🟡 | Fontos |
| 🟢 | Normál |

```markdown

| Téma | Kapcsolattartó | Megjegyzés |
|------|----------------|------------|
| [Terület] | **[Név]** | [Rövid leírás] |
```

```markdown

```

3. [MAGAS]       [Feladat]
4. [KÖZEPES]     [Feladat]

## Checklist Dokumentum Készítéshez

- [ ] BLUF szekció az elején
- [ ] Fő üzleti célok definiálva
- [ ] Objectives (max 6-8)
- [ ] Minden O-hoz Key Results (mérhető!)
- [ ] Minden O-hoz Initiatives (konkrét feladatok)
- [ ] Státuszok jelölve (emoji)
- [ ] Felelősök megnevezve
- [ ] Blokkolók kiemelve
- [ ] Kapcsolattartási mátrix
- [ ] Prioritási sorrend

---

## Tippek

1. **BLUF legyen rövid** - Max 1 képernyő
2. **Objectives legyen ambiciózus** - "Mit akarunk elérni?"
3. **Key Results legyen mérhető** - Szám, %, dátum
4. **Initiatives legyen konkrét** - "Ki, mit, mikor"
5. **Használj táblázatokat** - Könnyebb átlátni
6. **Státusz mindig friss** - Minden meetingen frissítsd
7. **Linkeld össze** - Ha egy Initiative másik O-hoz is kapcsolódik, jelöld (→ O6)

---

## Példa: Objective Kapcsolódások

Ha egy központi rendszer (pl. AI platform) több területet érint:

```markdown
> **Minden téma ide kapcsolódik** - a [Rendszer] integrálja:
> - O1 → [kapcsolat]
> - O2 → [kapcsolat]
> - O3 → [kapcsolat]
```
