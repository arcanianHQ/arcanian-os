### Estimated vs. instrumented

Mark estimates explicitly. A number without a real source is worse than no number — it signals confidence the system has not earned.

- **Instrumented:** value comes from a named source (Databox board, log file, counter in the task system). Shown as-is with a colour.
- **Estimated:** value is a gut number from the person filling this in. Prefix with `~` or write `estimate —` and set colour to ⚪ until instrumented.
- **Not yet instrumented:** row exists but no number can be produced. Leave value blank, colour ⚪, and name the commitment to install the instrumentation with a date.

A scorecard that hides estimates as if they were instrumented numbers is a scorecard that is already lying. Mark them, or remove the row.

## Starter numbers to pick from

Pick five to begin. Add up to fifteen over a year. Never more.

**Retention**
- Clients in month 1 / 6 / 12 / 18 (the 18-month cliff)
- Churn this month
- Clients renewed this month

**Commercial**
- MRR
- New engagements signed this month
- Average engagement value
- Proposals out / proposals accepted

**Delivery**
- Open projects
- Past-due deliverables
- Average time-to-close for projects that closed this week
- Open issues older than 14 days

**Learning**
- Patterns contributed this month (to `KNOWN_PATTERNS.md`)
- Post-mortems completed this month
- Reviews attended by practitioners

**Diagnostic rigour**
- Diagnoses completed this month
- Recommendations with measurable outcomes named
- Claims with source tags (as % of claims in shipped deliverables)

**Team**
- Practitioner hours on strategic vs. execution work
- Practitioner utilisation (% of billable capacity used)

**Client health**
- NPS-style signal (where collected)
- Last meaningful touchpoint per account
- Open escalations
